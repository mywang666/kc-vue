使用说明与交付流程

本页总结了本仓库的自定义 Agent、Skills、命令、模板与插件治理规则。目标是确保 需求一致性与交付完整性。

推荐流程

1. 输入准备 development-input-template.md + requirement-traceability-template.md
2. 开发与验证 /frontend 或终端命令 (tsc + build)
3. 输出交付 development-output-template.md + change-summary-template.md
4. 审查合并 frontend-review Agent + Git workflow

Agents

  • frontend：需求驱动交付与前端实现主力。
  • frontend-review：需求一致性审查与门禁。

Instructions 使用方式

  • 对 src 目录改动时自动加载约束。
  • 需求不清晰时默认最小实现并记录偏差。
当请求“新增页面”时，必须先提供需求条目与范围。
当请求“提交合并”时，必须符合 Conventional Commits。

Skills 使用方式

  • 以 /skill-name 调用流程化产出。
  • 输出包含覆盖矩阵与验证结果。
/requirement-driven-delivery
需求来源：PRD-2026-03
目标：列表筛选（仅需求内字段）

Commands 使用方式

  • 支持 /command-name 或终端执行。
  • 交付门禁按 Gate-1~Gate-5 执行。
/frontend-gate
npx tsc --noEmit
npm run build

Instructions 逐项说明

frontend.instructions 限制需求外 UI/交互扩展，只允许最小改动。 requirements-guard.instructions 强制需求条目与实现映射。 git-workflow.instructions 提交格式与 PR 合并要素约束。 large-project.instructions 需求/文件量大时强制分批交付。 
feat(order): 新增状态筛选
Batch-1 主流程 / Batch-2 守卫

Skills 逐项说明

requirement-driven-delivery 拆解需求并输出覆盖矩阵。 frontend-component 组件改造规范与接口设计。 frontend-testing 回归测试与风险点验证。 frontend-performance 性能定位与最小优化。 development-docs 输入/输出文档化。 
/frontend-component
组件：src/components/OrderList.vue
props/emits：显式类型

Commands 逐项说明

/frontend 开发与质量命令集合。 /frontend-gate 交付门禁流程。 /git-workflow 分支与提交规范。 /large-project 复杂项目分批计划。 
/git-workflow
feature/order-status-filter

Templates 逐项说明

development-input-template 需求来源、范围与风险记录。 requirement-traceability-template 需求到实现的映射矩阵。 development-output-template 交付与验证结果汇总。 change-summary-template 变更摘要与回滚说明。 pr-merge-checklist-template 合并前核对清单。

说明：新增下拉框 | 验证：手测 | 状态：完成

Plugins 逐项说明

Vue.volar Vue 语言服务与类型提示。 dbaeumer.vscode-eslint ESLint 代码检查。 esbenp.prettier-vscode 统一格式化风格。 path-intellisense 路径补全与快速定位。 gitlens Git 历史与 blame 信息增强。

Agent 逐项说明

@frontend 需求驱动实现与交付文档输出。 @frontend-review 需求一致性与门禁审查。 
@frontend 输出需求覆盖矩阵并标注未实现项
@frontend-review 检查是否存在需求外扩展

治理规则摘要

1) 只实现需求文档明确范围。2) 需求不清晰时走最小实现并记录偏差。3) 输出文档 必须包含验证结果与风险声明。
